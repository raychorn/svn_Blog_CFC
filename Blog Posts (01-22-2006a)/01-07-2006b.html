The Rabid CF Developer's BlogCFC 4.0.2 Enhancements Vol. #1

Okay, the time has come to deploy a few more enhancements for BlogCFC 4.0.2 - my hat is still quite off to the orginal author of this amazing Blogzilla.

Enhancement #1 - I give you all the concept of a "snapin". A "snapin" is a chunk of functionality that can be snapped-in just as one would snap together one's trousers each morning. In case you don't have snaps on your trousers or in case you don't wear trousers at-all just go ahead and think-up your own metaphor for this concept.

For the purposes of this incarnation of a snap-in there are four parts for each snap-in. JavaScript HEAD content that snaps into the HEAD of each document. The method for this part of each snap-in is called "jsCodeForHead". The second of the four parts is the JavaScript code that snaps into the document itself. The method name for this part of each sna-in is called "jsCode". The third part is the HTML content that one uses to launch into the snap-in when it comes time to actually use the functionality one wishes to snap into place. The method name for this part is called "htmlActionButton" because oddly enough the content that goes into this part tends to be some kind of button or widget or the like. The fourth part of the snap-in is the HTML code that supports the snap-in and the method for this part is called "htmlCode", oddly enough.

The reason for four parts rather than two or three or five or sixteen is the fact of how my first snap-in was created since it had four parts. Okay so my logical design lacks some imagination to be sure since it was driven by a real world need to add some functionality to an exiting piece of software for which I happened to have the source code.

My first snap-in:

The first snap-in I wanted to create was a simple enough JavaScript gizmo that would make it easier for me to turn program listings into Bloggable program listings.

The process of taking a raw code listing such as JavaScript or CFML code and turning it into something that can be posted to a Blog requires some small amount of engineering. All of the "<" and ">" characters must be turned into their funky-looking "&lt;" and "&gt;". Then all the whitespace is turned into "&nbsp;" meta-characters. And finally each line of code is placed on a separate row of a Table with a nice Table heading to make it easier to read. All this maintains the structure of the code being published while making it all look nicer than it might otherwise were it left to choose its own fate in life.

Take a look at this Snap-In in living color:

Notice the button labeled "[Format Code]". This button was created from the snap-in that resides in the file called "codeFormatterSnapIn.cfc".

This is what the snap-in looks like when the "[Format Code]" is clicked. Notice there are two panes. The pane on the left is where the raw source code is pasted. The pane on the right is where the formated code will be placed after the "[>>]" button is clicked. One closes this snap-in by pressing the "[X]" button.

Snap-Ins are automatically executed

The beauty of this concept is the fact that snap-ins are automatically executed at run-time. All the programmer need to is code a single CFC for each snap-in. All sna-ins are placed in the folder called "snapin".

Sample Snap-In CFC

Notice the CFC one must extend which is called "snapIns". You can read the rest of the code below to get a sense of how to create your own snap-ins for the version of the BlogCFC 4.0.2 I have posted to the Downloads section of my Blog.

The code that installs the Snap-Ins

Notice there are four parts shown below and notice also each of these four parts maps to one of the four parts mentioned previously in this Blog Post.

Snap-In Activation Code (layout.cfm) HEAD Code
 <link rel="alternate" type="application/rss+xml" title="RSS" href="#application.rooturl#/rss.cfm?mode=full" /> 
 <cfscript> 
  for (i = 1; i lte application.snapIns.count; i = i + 1) { 
   try { 
    if (IsCustomFunction(application.snapIns.snapIns[i].jsCodeForHead)) { 
    // application.snapIns.snapIns[i].init(); 
     writeOutput(application.snapIns.snapIns[i].jsCodeForHead()); 
    } 
   } catch (Any e) { 
   } 
  } 
 </cfscript> 

Snap-In Activation Code (layout.cfm) JavaScript Code
  <cfscript> 
   for (i = 1; i lte application.snapIns.count; i = i + 1) { 
    try { 
     if (IsCustomFunction(application.snapIns.snapIns[i].jsCode)) { 
      writeOutput(application.snapIns.snapIns[i].jsCode()); 
     } 
    } catch (Any e) { 
    } 
   } 
  </cfscript>
