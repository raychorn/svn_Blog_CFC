Typically JavaScript code is delivered to a web browser in the form of a file, a source file. The JavaScript source file is then downloaded to the browser where it will be cached for later use. Caching is both a good and bad thing. Caching is a good thing because once the file is cached it will speed the web app by keeping the already cached files from being downloaded again and again. Caching is a bad thing because people can easily view the contents of all cached files thus allowing them to read your source code to see how you are doing things. Sure it is possible to obfuscate JavaScript code to make it more difficult for people to read the source files but obfuscation is not the best way to keep your JavaScript code safely hidden from view. The best way to keep your JavaScript source code hidden from view is to use RABID_AJAX™ to keep your JavaScript from ever getting to the browser's cache.

RABID_AJAX™ will use its own caching method to reduce the bandwidth requirements for your web app as it is being served to your site's visitors. Personally, I would rather use a little more bandwidth to keep my valuable JavaScript hidden from view, if I had the option of doing so.

RABID_AJAX™ will also make it possible to deliver web apps that use ColdFusion or any other CGI language via .html files rather than .cfm files or .php files or the like. Oh sure, you could use the Apache web server and setup a URL Rewrite Rule that hides the fact that you are using ColdFusion or PHP while allowing search engines to be optimized through the use of static-looking URLs that don't use URL parameters - you should still do this anyway even when using RABID_AJAX™.

RABID_AJAX™ makes it possible to completely hide the underlying server-side systems from view by making it possible to use nothing but .html files that have nothing but HTML in them. Some people have found that they can put their ColdFusion code into .html files which can also make it look like ColdFusion is not being used however this is not the same thing as actually hiding ColdFusion from view, for instance.
