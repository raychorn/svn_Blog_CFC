WTF did you just say ?!?

Any non-Admin function such as adding Trackbacks or Comments allows for cross-domain anonymous data entry with absolutely no resistance from BlogCFC.

When we brought this to Ray Camden's attention his response was to say that Admin functions require the Admin Role to access functions that store data in a database in BlogCFC. Okay, we dont dispute this - yes, Admin functions do require the Admin Role so those functions are safe from hacking or cross-domain attacks. But, the two non-Admin functions that DO allow cross-domain hacking are "addTrackBack()" and "addComment()" and this can be easily verified by looking carefully at the source code found in Blog.cfc which is typically stored in org/camden/blog/blog.cfc .

How-To Verify that these Backdoors exist and that YOUR site is at-risk ?!?

Do a search in the file named Blog.cfc for any references to "CGI.HTTP_". If you find any references to CGI.HTTP_REFERER this would be a good thing because these cross-domain hacking attacks could be eliminated by a small amount of code as follows:

<cfif (FindNoCase(CGI.SERVER_NAME, CGI.HTTP_REFERER) eq 0)> 
 <cflocation url="index.cfm">  <!--- this URL depends on the actual location of index.cfm relative to the l
ocation of this code ---> 
</cfif> 

The code above does one thing very very well. It ensures nobody can access certain functions from using cross-domain techniques. Very simply put, anytime the CGI.HTTP_REFERER does not contain the CGI.SERVER_NAME the visitor would be redirected back to the "index.cfm" page. Some of you may not be aware that it is possible to use winInet to program an anonymous POST that does not have any REFERER which from the perspective of the CF Server the CGI.HTTP_REFERER would be blank as opposed to a POST coming from within your own domain when the CGI.HTTP_REFERER would contain a substring that matches CGI.SERVER_NAME.

What is "passive Pfishing" ?

We all know what Pfishing is, right ? The use of illegal means to obtain email address right from your web server or browser.

Passive Pfishing is a similar technique but it uses passive means to obtain email addresses.

Whenever someone Subscribes to a BlogCFC site the BlogCFC software will send an email to those people who have submitted their email address as a Subscriber. Along with each of those emails BlogCFC also sends the email address of the person who originated the Comment or TackBack. This means BlogCFC is passively Pfishing YOUR database for email addresses that it sends to the Subscriber. The person using this passive Pfishing technique would only need to sit back and watch the emails pour-in with email addresses that could be used to market back to real people who posted comments at YOUR Blog site without your knowledge or conscent.
