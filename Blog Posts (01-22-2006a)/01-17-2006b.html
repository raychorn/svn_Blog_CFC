This took several days due to my workload and my need to figure-out how to code a URL Rewrite expression with nearly no examples to use.

As it turns out BlogCFC uses an arcane undocumented method of passing parameters which follows the format as follows:

blog-url/index.cfm/year/month/day/alias

The problem is that Apache 2.0.55 doesn't know what to do this this type of URL using the default configuration that comes out-of-the-box. I have no doubt the brilliant King Camden knows how his web server is setup and for some odd reason my 64-bit Turion laptop's Apache 2.0.54 knows how to handle this URL but Apache 2.0.55 does not.

So I had to read-up and do some research to figure out an appropriate URL Rewrite rule to allow me to translate the original design from BlogCFC's arcane URL format to one I like better since I don't want anyone to know about index.cfm when it comes time to click on links for Blog Posts and such.

Now you geniuses out there have to forgive me because I am more than just a little stupid when it comes to regular expressions and such although I am sure King Camden would know this one for sure.

The regular expression I settled on looks something like this: "[0-9A-Z]+[\-][0-9A-Z]+[\-][0-9A-Z]+[\-][0-9A-Z]+" and it seems to handle the GUID's that are used to form the entry ID for blog posts. Like I said, I am more than a little stupid about regular expressions at this point so I am sure there is a more elegant way to code this silly expression. Just in case my hunch is right and there is a more elegant way to code the expression, keep it to yourself - I don't want to hear from you RE geniuses out there because I got mine working - if my expression breaks then I'll put on my dunce cap and work real hard to fix it albeit in my stupid manner.
