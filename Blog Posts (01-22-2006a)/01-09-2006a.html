The Rabid CF Developer has done it again ! Pods without Borders for BlogCFC 4.0.2

BlogCFC 4.0.2 is a reasonable Blog Server but it lacks some pazazz in the look-n-feel department.

Today I give you Pods without Borders - no this is not a humanitarian effort to streat Pods across the globe but it is a way to allow Pod Layouts to be rendered without the Borders to allow for some variety in the look-n-feel of your friendly neighborhood Blog Pods..

Notice the attributes.containerClass - this effectively makes the containerClass attribute for this custom tag into a variable parameter that can be used or not used which means the Pod can have a border from the CSS class one uses for the podLayout so that if the containerClass is made blank by passing in an empty string the pod will not have a border or you can just as easily make a new CSS class that sans the border while maintaining the rest of the CSS parameters such as font-size or the like.

podlayout.cfm
<cfsetting enablecfoutputonly=true> 
<cfprocessingdirective pageencoding="utf-8"> 
<!--- 
 Name         : podlayout.cfm 
 Author       : Raymond Camden  
 Created      : October 29, 2003 
 Last Updated : July 22, 2005 
 History      : PaulH added cfproc (rkc 7/22/05) 
 Purpose   : Pod Layout 
---> 
 
<cfparam name="attributes.title"> 
 
<cfif thisTag.executionMode is "start"> 
 
 <cfparam name="attributes.containerClass" type="string" default="rightMenu"> 
 
 <cfoutput> 
 <p> 
 <cfif (Len(attributes.containerClass) gt 0)> 
  <div class="#attributes.containerClass#"> 
 <cfelse> 
  <div> 
 </cfif> 
 <cfif (Len(attributes.title) gt 0)> 
  <div class="menuTitle">#attributes.title#</div> 
 </cfif> 
 <div class="menuBody"> 
 </cfoutput>   
 
<cfelse> 
 
 <cfoutput> 
 </div> 
 </div> 
 </p> 
 </cfoutput> 
 
</cfif> 
 
<cfsetting enablecfoutputonly=false>
